# `parserast.cpp`

## Abstract Syntax Tree (AST) & Token Ordering

`parserast.cpp` takes the flattened vector of token strings generated by the lexer and organizes them into a structured execution sequence. It emulates an Abstract Syntax Tree (AST) by evaluating the tokens recursively according to standard mathematical precedence Rules (**PEMDAS**). The final output is an ordered instruction vector containing broken-down three-address code steps with temporary placeholder tokens (e.g., `T1`, `T2`).

---

## The Parser Logic

The entry-point function `parserast()` receives the tokenized vector, computes the total number of operators using a utility counter, initializes a global assignment counter (`opCounter = 1`), and hands execution off to the core `pemdas()` function.

The parsing and reduction process inside `pemdas()` relies on two core phases:

### 1. Parenthetical Sub-Expression Extraction
Before handling naked mathematical operators, the algorithm iteratively isolates expressions wrapped inside parentheses:
* **Inner-Most Isolation**: The function scans the vector from right to left to find the deepest open parenthesis (`(`), then scans from that index to the right to locate its matching closing parenthesis (`)`).
* **Recursive Reduction**: It clips out the sub-expression between these bounds and recursively feeds it back into `pemdas()`. 
* **Placeholder Replacement**: 
  * If the sub-expression contains operators, its structural steps are merged into the main tracking vector. The sub-expression is replaced in the root token sequence by its final generated temporary token (`T_n`).
  * If no operators exist inside (e.g., a single isolated integer `(4)`), it is treated as a plain value, and the parenthesis are simply stripped away.

### 2. Multi-Pass Operator Precedence Reduction
Once parenthetical segments are neutralized, the function processes the remaining tokens across 5 strict hierarchy tiers:

| Precedence Tier | Operators/Functions Handled | Evaluation Pattern |
| :--- | :--- | :--- |
| **Tier 0** | `sin`, `cos`, `tan`, `sec`, `csc`, `cot`, `log`, `ln` | Right-side tracking only (Unary prefix) |
| **Tier 1** | `^` (Exponents) | Left and Right tracking (Binary infix) |
| **Tier 2** | `*`, `/` (Multiplication & Division) | Left and Right tracking (Binary infix) |
| **Tier 3** | `+`, `-` (Addition & Subtraction) | Left and Right tracking (Binary infix) |
| **Tier 4** | `=` (Assignment) | Left and Right tracking (Binary infix) |

* **Left-to-Right Execution**: For each tier, a `while` loop checks if any matching operators exist. It picks the first available operator from the left to break ties dynamically (e.g., between `*` and `/`).
* **Operand Isolation**:
  * **Binary Operators**: Grabs the elements directly to the left (`tieBreaker - 1`) and right (`tieBreaker + 1`) of the operator index.
  * **Unary Operators (Functions)**: Skips left-side collection and sets `leftValue = "0"`, consuming only the target right-side argument.
* **Tree-Flattening String Generation**: 
  The isolated sequence is stripped out of the main array, bundled into an abstract assignment triplet, and appended to the final instruction vector in this structural form:
  $$\text{[Left Value]} \rightarrow \text{[Operator]} \rightarrow \text{[Right Value]} \rightarrow \text{[->]} \rightarrow \text{[T\_n]}$$
  The original expression string slice is then physically replaced in-place inside `inputVector` by the `T_n` token so subsequent tier loops can treat it as a single numeric operand.

### Example Output Transformation
An input vector representing: `3 * x + 4 / 3 - 1`
Will be processed, evaluated, and compiled into the following sequential output string sequence:
`[3], [*], [x], [->], [T1], [4], [/], [3], [->], [T2], [T1], [+], [T2], [->], [T3], [T3], [-], [1], [->], [T4]`

## Visual Execution Pipeline


```mermaid
graph TD
    Start([parserast Called]) --> DebugIn[If debugMode: Print Initial Token Vector]
    DebugIn --> CountOps[Calculate initial operatorCount via utility]
    CountOps --> CallPemdas[Invoke pemdas inputVector, operatorCount, opCounter=1]
    
    %% Inside PEMDAS Parenthesis Check
    CallPemdas --> ParenCheck{Does inputVector contain open parenthesis?}
    ParenCheck -- Yes --> FindInner[Scan Right-to-Left for deepest open parenthesis<br>Then from there scan Left-to-Right for matching closed parenthesis]
    FindInner --> SliceSub[Extract inner elements into subExpression]
    SliceSub --> Recurse[Recursively call pemdas on subExpression]
    
    Recurse --> SubEmpty{Is subResults empty?}
    SubEmpty -- Yes --> PlainNum[tempToken = subExpression 0 ]
    SubEmpty -- No --> GetTemp[tempToken = Last generated T_n from subResults]
    
    PlainNum --> MutateInput[Erase subExpression + parentheses from inputVector]
    GetTemp --> MutateInput
    MutateInput --> InsertTemp[Insert tempToken back at open parenthesis index]
    InsertTemp --> MergeResults[Append subResults components to orderedVector]
    MergeResults --> ParenCheck

    %% Tier Precedence Processing
    ParenCheck -- No --> PrecedenceLoopStart[Initialize Priority Tiers:<br>Tier 0: Functions -> Tier 4: '=']
    
    PrecedenceLoopStart --> LoopTiers{Iterate Tier n from 0 to 4}
    LoopTiers -- Loop Continues --> ContainsCheck{Does inputVector contain<br>any operator from Tier n?}
    
    %% Tier Evaluation In-Loop
    ContainsCheck -- Yes --> LeftmostOp[Scan Left-to-Right: Find first matching operator index tieBreaker]
    LeftmostOp --> OpTypeCheck{Is operator a Unary Function<br>sin, cos, log, etc.?}
    
    OpTypeCheck -- Yes --> UnaryPrep[leftValue = '0'<br>rightValue = inputVector tieBreaker + 1 <br>hasLeftValue = false]
    OpTypeCheck -- No --> BinaryPrep[leftValue = inputVector tieBreaker - 1 <br>rightValue = inputVector tieBreaker + 1 <br>hasLeftValue = true]
    
    UnaryPrep --> GenToken[Generate new temporary token T_n<br>Increment opCounter]
    BinaryPrep --> GenToken
    
    GenToken --> PushOrdered[Append to orderedVector:<br>leftValue -> op -> rightValue -> '->' -> T_n]
    
    PushOrdered --> VectorErase{hasLeftValue == true?}
    VectorErase -- Yes --> EraseBinary[Erase 3 tokens: left, op, right<br>Insert T_n at tieBreaker - 1] --> ContainsCheck
    VectorErase -- No --> EraseUnary[Erase 2 tokens: op, right<br>Insert T_n at tieBreaker] --> ContainsCheck
    
    ContainsCheck -- No --> LoopTiers
    
    %% Exit paths
    LoopTiers -- Loop Done --> ReturnPemdas[Return orderedVector]
    ReturnPemdas --> DebugOut
